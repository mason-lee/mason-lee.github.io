Introduction

Listy allows families and roommates to organize and distribute list items among themselves. Functional prototyping is implemented using Javascript framework Framer.js.

About the application

Arranging meals for the week, preparing the house for a birthday party, or planning equipment for a camping trip with friends; these are all instances where Listy can be handy. Common to all these events are the following tasks:

  1. Writing a list of food and/or items that will be needed.
  2. Collecting the items as a group, often dividing who buys what.
  3. Making last minute changes such as adding or removing list items.

Process

After a series of sketching sessions, I came up with Axure wireframes in order to focus on user interface behaviors, interactions, and visual elements throughout different pages. I then tested the design with my roommates, because they are the people that would most likely organize the lists.

With the participants, I first gave them a short introduction of the application - "Listy is a utility application which is designed to allow families and roommates the ability to organize and distribute list items among themselves....". I decided not to give them a demo of the app because I primarily wanted to test the intuitiveness of the navigation systems. Two main tasks given to participants were creating an event and add list items.

Result

There were 3 main problems discovered from the test:

  1. Event list and item list (lists that are part of an event) were confusing.
  2. Navigation system was unintuitive- participants didn't easily recognize the "+" button.
  3. Checked items cluttered the screen

Based on these main findings, I redesigned the user interface of the application by completely changing the navigation system and list organizations.

Redesigned Elements

  1. The only navigation item is a list of events.
  2. Words are used for buttons instead of icons - eg. "All Events", "add", "Done"
  3. Checked items are greyed so that users can focus more on unchecked items.
  4. Individual event page includes a list of items, whereas in the previous design, clicking an event led users to a different page that has a list of items under that event.
  5. Images are used with event description in order to make an event page feel more personal and enjoyable.
